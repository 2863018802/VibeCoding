# 520 表白网站计划

## 📅 更新时间
2026年5月19日（手机端重构版）

---

## 🎯 目标
制作一个浪漫的表白网站，在520当天送给女朋友。
**设备优先：手机竖屏**（桌面端作为次要兼容）

---

## 🎨 整体设计理念

**核心理念：手机是亲密的信纸，不是展示型的网页。**

桌面端思维：横向大图、多栏布局、鼠标悬停交互
手机端思维：竖向全屏滑动、单指滑动切换、单手操作

> 📷 *照片墙点缀：第1屏左上角和右下角各放一张陈奕迅小照片，倾斜5°~15°，像随手贴在墙上的拍立得，大小约 60×60px，不遮挡内容，仅做氛围装饰*

## ✨ 叙事结构：竖向全屏章节式（5屏）

每屏 = 100vh，用 swipe 或竖向滑动切换，沉浸式无导航条。

```
[第1屏] 启动仪式
 ↓ 点击/滑动
[第2屏] 陈奕迅歌词墙
 ↓ 滑动
[第3屏] 情书 + 相册
 ↓ 滑动
[第4屏] 回忆问答
 ↓ 滑动
[第5屏] 结尾：爱情证书 + 未来愿望
```

---

## 📱 每屏详细设计

### 第1屏：启动仪式（Entry）

- 全屏深紫黑背景 + 淡淡的心跳光晕
- 顶部：在一起天数（超大数字，52px，加粗，字间宽字）
- 中部：陈奕迅歌词片段（打字机效果）
  > *"除非你是我，才可与我常在。"*
- 底部：一个巨大的「❤️ 轻轻点我」按钮（80px 圆形，脉冲动画）
- **点击按钮时**：触发背景音乐自动播放 + 淡入切换到第2屏

**技术要点：**
- `Audio` 对象绑定在点击事件上，移动端可正常播放
- CSS `@keyframes` 光晕脉动，不用 Canvas 降低性能
- 按钮用 `touch-action: manipulation` 防重复触发

---

### 第2屏：陈奕迅歌词墙（歌词即是视觉）

- 背景：深色模糊（演唱会舞台视觉）
- 歌词逐行从两侧淡入，配合舞台光效
- 精选歌词：
  > *"从你的故事里，我终于读懂了为什么爱一个人会心碎。"*
  > *"我知，日后，路上或没有更美的邂逅。"*
  > *"若只是喜欢，何必夸张成爱。"*
- `IntersectionObserver` 监测歌词行进入视口，触发淡入动画
- 背景用 CSS `radial-gradient` 模拟舞台聚光灯

> 📷 *照片墙点缀：歌词墙背景左上角放一张陈奕迅侧脸小照（45×45px，旋转-8°），右下角竖放一张演唱会上拿麦克风的小照（40×70px，旋转12°），中间歌词段落右侧留白处贴一张小照片（50×50px，旋转5°），均做高斯模糊处理融入背景，opacity 约 0.3~0.5，像舞台上若隐若现的影子*

---

### 第3屏：情书 + 相册（私密感）

**上半：情书文字区**
- 娇娇的名字淡入
- 情书内容打字机效果（每分钟40字，模拟手写信节奏）
- 字体：`Georgia, serif`（有温度的手写感）
- 行高 1.9，字间距宽松

**下半：相册入口**
- 一张代表合照作为封面
- 标签：「我们的192天 📸 点击翻阅」
- 点击展开：全屏竖向轮播图（纯 CSS + JS）
- 支持左右滑动切换、点击关闭

**技术要点：**
- 相册用 `overflow-x: scroll` + `scroll-snap-x` 实现原生滑动
- `touch-action: pan-y` 避免横滑与页面滑动冲突
- 图片懒加载：`loading="lazy"`

---

### 第4屏：回忆问答（问答互动）

- 标题：「关于我们的6道题」
- 题目用卡片展示，每次只显示一道题
- 点击选项，选项变色反馈
  - 正确：绿色 + 心跳弹跳动画 + 继续下一题
  - 错误：红色抖动 + 「再想想？」提示
- 全部答对：屏幕碎裂效果（CSS） + 显示礼物彩蛋页面

**题目列表：**
1. 我们是哪一天在一起的？→ `2025.11.09`
2. 睿睿的生日是什么时候？→ `2002.02.17`
3. 睿睿现在做什么工作？→ `人工智能应用开发`

**彩蛋奖品：** 腰部按摩仪 🎁

> 📷 *照片墙点缀：问答卡片区域左上角贴一张陈奕迅演唱会的远景照片（小图 40×40px，旋转-10°），右下角放一张特写麦克风的小照（35×55px，旋转8°），作为答题过程中的氛围点缀，不干扰答题操作*

**技术要点：**
- CSS `transform: translateX` 做题目切换，无刷新
- 正确/错误反馈用 CSS class toggle + 短暂 setTimeout
- 彩蛋页用 CSS `clip-path` 模拟碎屏

---

### 第5屏：结尾（爱情证书 + 未来愿望）

**模块A：爱情证书**
- 日期：「2025年11月9日 — ∞」
- 两人名字：「睿睿 ❤️ 娇娇」
- 一句誓言：「除非你是我，才可与我常在。」
- 底部：「颁发日期：2026年5月20日」
- 旋转唱片图标（CSS animation）

> 📷 *照片墙点缀：证书旁边散落 3 张陈奕迅小照片——一张正面微笑小照（55×55px，旋转-6°），一张拿着吉他弹唱的小照（45×65px，旋转10°），一张演唱会灯光下的剪影小照（50×50px，旋转-12°），配合金色香槟色边框，呼应纪念日主题*

## 🎤 陈奕迅视觉语言专项

不只是音乐，而是「演唱会体验」：

| 元素 | 实现方式 |
|------|---------|
| 舞台光 | CSS `radial-gradient` 多层叠加，模拟追光 |
| 灯海效果 | 小圆点粒子动画，像演唱会的手机灯海 |
| 歌词视觉化 | 歌词拆成单字，每字独立淡入，字体渐变 |
| 唱片机元素 | 结尾页加旋转唱片图标（CSS animation） |
| 暗场氛围 | 大量留黑，呼吸感强 |
| 手写字 | 关键文字用 `Georgia, serif` + 字间 |
| 照片墙点缀 | 散落小照片（40~60px），轻微旋转（-12°~+12°），低透明度（0.3~0.6），高斯模糊，像贴在墙上的拍立得 |

## 🛠 技术实现方案

**栈：纯 HTML + CSS + Vanilla JS**（单文件，可直接部署）

| 模块 | 实现方式 |
|------|---------|
| 全屏章节 | CSS `scroll-snap-type: y mandatory` + `height: 100svh` |
| 音乐播放 | 点击事件触发 `audio.play()`，解决移动端限制 |
| 爱心雨 | 少量 CSS `@keyframes` 粒子，不用 Canvas（省性能） |
| 相册轮播 | CSS `scroll-snap` + `overflow-x: scroll` |
| 打字机 | JS `setInterval` 逐字插入 |
| 问答逻辑 | JS 状态机，class toggle 控制显示/隐藏 |
| 碎屏彩蛋 | CSS `clip-path` + 多层 `div` 模拟碎片 |
| 灯海效果 | 50-80个 `<span>` 小圆点，各自 `@keyframes` 随机飘动 |
| 照片墙装饰 | `<img>` 绝对定位散落各屏角落，`transform: rotate() scale()`，`filter: blur() opacity()`，用 `z-index` 放内容层下方 |
