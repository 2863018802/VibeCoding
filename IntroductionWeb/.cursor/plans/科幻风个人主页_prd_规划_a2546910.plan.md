# 科幻风个人主页 PRD 规划

## 1. 项目概述

- **项目名称**：黎潇说明书
- **定位**：科幻风格的个人作品展示网站
- **目标用户**：访客（面试官、同行、技术社区）

## 2. 核心功能模块

- **首页**：科幻风 Hero 区域 + 个人简介
- **博客**：文章列表 + 文章详情页，支持 Markdown
- **项目经验**：项目展示卡片 + 详情页
- **关于**：个人履历、CS 技能树（本科 C/Java/Python/数据结构/MySQL + AI 领域 ML/DL/RAG/Agent + 工程能力 FastAPI/Docker/K8s/GitHub Actions）+ 政治面貌、学生工作经历

## 3. 技术选型

- **前端框架**：Next.js 14+（React 生态，SSR 支持，静态导出友好）
- **样式**：Tailwind CSS + 自定义 CSS 变量（科幻光效）
- **内容管理**：Markdown 文件存储（/content 目录）
- **部署目标**：腾讯云服务器（已有）
- **CI/CD**：GitHub + GitHub Actions 自动构建部署

## 4. GitHub Actions 部署流程

1. 开发者 push 代码到 GitHub 仓库
2. GitHub Actions 自动触发：
  - 安装 Node.js 环境
  - npm install 依赖
  - npm run build 构建
  - 通过 SSH 将构建产物同步到腾讯云服务器
  - Nginx 托管静态文件

## 5. 域名 & SSL

- 域名解析到腾讯云服务器
- Nginx 配置 HTTPS（Let's Encrypt 免费证书）
- 可选：七牛云/又拍云 CDN 加速

## 6. 页面结构（路由）

- `/` — 首页
- `/blog` — 博客列表
- `/blog/[slug]` — 博客详情
- `/projects` — 项目经验
- `/honors` — 成果与荣誉墙
- `/knowledge` — 知识体系图
- `/about` — 关于我

## 7. 内容创作流程

1. 在 `/content/blog`、`/content/projects` 或 `/content/honors` 添加 Markdown 文件
2. push 到 GitHub
3. CI/CD 自动构建并发布

## 8. 开发任务拆分

1. 项目初始化（Next.js + Tailwind）
2. GitHub 仓库创建 + GitHub Actions 配置
3. 基础页面框架 + 导航栏 + 主题切换
4. 首页 Hero 区域（科幻动效）
5. 博客模块
6. 项目经验模块
7. 关于页面（含荣誉墙入口 + 知识体系入口）
8. 成果与荣誉墙模块
9. 知识体系图模块
10. CI/CD 部署测试
11. 域名解析 + HTTPS 配置

## 9. 求职定位 & 特色模块

**目标**：面向央国企技术岗，展现「基本功扎实 + 规划清晰 + 文档能力强」的形象。

**特色强化模块：**

- **成果与荣誉墙** `/honors`：奖学金、竞赛奖项、学术论文、软著专利、英语成绩等，以「成就解锁」形式呈现
- **知识体系图** `/knowledge`：结构化知识树/星系图，展示系统性学习能力（按本科 CS 基础 + AI 技术领域组织：C/Java/Python/数据结构/MySQL → ML → DL → RAG → Agent）
- **关于页面**：`关于` 中融入政治面貌（党员/预备党员加分项）、学生工作经历，体现综合素质与稳定性
- **主题切换**：深色科幻风 + 浅色护眼模式，服务不同阅读场景（面试官办公室灯光）

### 10.7 技术栈展示设计方案（科幻并列结构）

摒弃树形隐喻，改为**四维数据港**——四个平行能量管道横贯全息界面，每个管道代表一个技术层，从左到右对应：界面层 → 逻辑层 → 智能层 → 基础设施层。

整体视觉布局：

```
┌──────────────┬────────────────────┬───────────────┬──────────────┐
│  🎨 前端      │  ⚡ 后端          │   🔮 AI 引擎   │  🛡️ 运维     │
│              │                    │               │              │
│  Next.js     │  Python            │  ML·Scikit    │  Docker      │
│  React       │  C / Java          │  PyTorch      │  K8s         │
│  Tailwind    │  FastAPI           │  大模型·DeepSeek│  GitHub Actions│
│              │  REST API          │  ·Qwen·Ollama │  LangSmith   │
│              │  Streaming         │  Agent·LangChain│  腾讯云       │
│              │  MySQL             │  ·Dify·Coze   │              │
│              │                    │  RAG·Faiss    │              │
│              │                    │  ·Milvus·GraphRAG│            │
│              │                    │  数学·数值分析  │              │
│              │                    │  ·数理统计     │              │
│              │                    │  ·矩阵论       │              │
└──────────────┴────────────────────┴───────────────┴──────────────┘
```

**四根管道视觉设计：**

- **🎨 前端**（蓝色光流）：界面输出层，管道入口有代码粒子效果
- **⚡ 后端**（橙色光流）：核心逻辑层，管道内有数据脉冲动画
- **🔮 AI 引擎**（紫色光流，最宽）：智能核心层，管道最粗，内含 ML / 大模型 / Agent / RAG / 数学五个节点群，节点间有能量连线
- **🛡️ 运维**（绿色光流）：基础设施层，管道内有链路追踪光点

**动画效果：**

- 四根管道内各有**独立颜色的光流**从左向右涌动，象征数据在各层间流转
- 悬停任一节点，该节点会**发光放大**，显示具体技术栈 tooltip
- 页面加载时，四根管道从左到右**依次展开**，像数据流依次流经各层
- 每根管道外框用 **scan-line 扫描线效果**，模拟科幻终端界面
